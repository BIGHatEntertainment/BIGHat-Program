## Setup wizard is now guaranteed to appear before first login

Prior versions could silently skip the Setup Wizard when the React app
couldn't reach `/api/native/info` — the fail-open default treated the
machine as a generic webapp, dropped the user at `/login`, and login
predictably failed because no master admin existed yet. v31.0.11
makes that impossible:

* `NativeContext.refresh()` now retries the backend up to 5 times with
  exponential backoff (the launcher takes 2-3 seconds to start the
  backend, so the first React fetch sometimes races the listener).
* If all retries fail AND this is a native install (the build was
  compiled with `REACT_APP_BACKEND_URL=""`), the app now renders a
  clear "BIG Hat can't reach its background service" screen with a
  Retry button. Customers will never again be silently funneled to
  the login form when setup hasn't run.

## Includes everything from v31.0.10

* Installed app now uses relative API URLs (fixes the "LOG IN TO 127"
  Google screen + the always-failing password login).
* Hardcoded employee passwords removed from source. Env-driven seeding.

* Regression test blocks credentials from sneaking back into source.
