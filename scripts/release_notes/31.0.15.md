# BIG Hat Entertainment v31.0.15

## Critical fix: blank-window incident root cause

v31.0.14 still showed a blank dark-blue window on launch. The DevTools
console screenshot from a customer install pinpointed the true cause:

```
Uncaught ReferenceError: Cloud is not defined
    at $Q (SetupWizard.jsx:573:24)
```

`SetupWizard.jsx` used a `<Cloud />` icon (lucide-react) that was never
imported — React threw immediately on mount, leaving the window empty.

### Why it slipped through v31.0.14

`frontend/craco.config.js` had overridden ESLint to ONLY enable
`react-hooks/recommended`, silently dropping `react/jsx-no-undef` and
`no-undef`. The production build emitted bad JS without ever warning.

A second instance of the same bug class (`locationName` used in a
`catch` after being declared in a `try`) was hiding under the same
blind spot. Both are fixed in v31.0.15.

### What changed

- Added the missing `Cloud` icon import to `SetupWizard.jsx`.
- Hoisted `locationName` to function scope in `PresentationMode.jsx`
  so the `catch` branch can reference it.
- Pinned `no-undef` and `react/jsx-no-undef` as hard ESLint errors in
  `craco.config.js`. Future missing imports now fail the build instead
  of shipping to customers.
- New pytest `test_frontend_no_undef.py` runs the same check offline
  and self-verifies (it caught both bugs).
- NSIS installer now wipes the old `backend/static/` subtree before
  laying down the new bundle. Stale hashed JS files (~3 MB per release)
  no longer accumulate across upgrades.

stable). The next major version is the **Tauri** native shell that

the first signed builds will publish on tag `v32.0.0`.
