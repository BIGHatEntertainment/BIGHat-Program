## Critical fix: customers' installed app couldn't authenticate

The v31.0.5–v31.0.9 builds had a fatal bug: the React frontend was compiled
with `REACT_APP_BACKEND_URL` baked-in to point at the dev preview
environment, not to the customer's own machine. On every installed copy:

* Google sign-in still appeared (the native-mode hide flag silently
  failed because `/api/native/info` was unreachable) → users landed on
  the "LOG IN TO 127" Emergent OAuth screen.
* Email/password login returned "Authentication failed" because the
  login request was hitting our preview server, not the customer's
  embedded backend, and our preview doesn't know their credentials.

**v31.0.10 fixes this**: the installer build now compiles the frontend
with relative API URLs (`/api/...`), so the app talks to its own
embedded backend at `127.0.0.1:8001`. Login + Setup Wizard + native
Google-hide all work as designed.

## Security hardening

The previous source had a default employee password literal hardcoded
in 11 places in `server.py` (visible in the public GitHub repo), plus
the dev `system_config.json` with a bcrypt hash of the test password.
This release:

* Replaces all hardcoded passwords with env-driven constants
  (`DEFAULT_HOST_PASSWORD`, `ADMIN_MASTER_PASSCODE`, `SEED_PW_*`).
  If env vars are unset at first boot, the server generates random
  passwords and logs them ONCE for the operator to write down.
* Removes the `default_password` field from the
  `/api/host/password/is-default/{id}` response (it used to literally
  return the default).
* Adds the `is_default_password` flag to the `/api/host/login`
  response so the client never has to compare against a baked literal.
* Removes `backend/native/system_config.json` and `backend/static/`
  from git tracking — both contained per-install secrets / build
  artifacts and should never have been there.
* Adds a `pytest` regression test (`test_no_plaintext_credentials.py`)
  that fails CI if any future change reintroduces a known-leaked
  literal or a bcrypt hash into tracked source.

## Important: macOS Gatekeeper

The Mac builds are **unsigned developer previews** (no Apple
Developer ID enrolled yet). First-run Mac users will see "BIG Hat
Standalone is damaged and can't be opened". Bypass with:

```bash
xattr -dr com.apple.quarantine "/Applications/BIG Hat Standalone.app"
```

Signed + notarized builds will replace these once the Apple Developer
ID is provisioned. No re-purchase required.
