## Critical fix: blank window on launch

**Symptom**: After installing v31.0.13 and double-clicking the BIG Hat
desktop shortcut, the chromeless window opened but the page stayed
solid deep-blue with no UI.

**Root cause**: The launcher `start_bighat.vbs` was using Microsoft
Edge's `--app=URL` chromeless mode. This was explicitly forbidden in
the v31.0.6 NEVER-DO RULES (after a similar incident) but had been
re-introduced since. The `--app=` mode hides the JavaScript console
(no F12 dev tools by default), so any runtime issue inside the
chromeless window renders as a blank page with no diagnostic.

**Fix**: Reverted `start_bighat.vbs` to the user-approved path from
v31.0.5 — launch the user's default browser via
`WshShell.Run TARGET_URL, 1, False`. The app now opens in a normal
Chrome/Edge/Safari tab where the user can see what's happening and
use DevTools if needed.

**Guardrail**: Added `backend/tests/test_launcher_vbs_contract.py`
with four assertions that fail if any future change reintroduces
`--app=`, `msedge.exe` lookup, `chromiumExe` plumbing, or
`pywebview`. This regression cannot ship again without explicitly
deleting the test (which would itself bounce in code review).

## Installer integrity check

The publish script now refuses to upload a truncated .exe (runs
`7z t` before HTTP PUT). The v31.0.13 "Installer Integrity Check has
failed" error cannot recur.
