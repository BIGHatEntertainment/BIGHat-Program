## What's new since v31.0.6

* **First-install Setup Wizard now actually runs.** Previously the
  installer shipped a dev seed config that short-circuited the wizard
  and made login impossible. v31.0.7+ correctly prompts every new
  customer to create their master admin and enter their license key.
* **Google sign-in is hidden on native installs.** Eliminates the
  "LOG IN TO 127 / Emergent" window-title issue when the chromeless
  browser tried to navigate to the OAuth provider. Email/password is
  the only auth method on desktop; Google login stays available on the
  webapp.
* **License activation is now bound to the cloud at first run.**
  Bad / fake keys are rejected up front. Offline installs complete in
  a `pending_cloud_activation` state and silently finish activating
  when the network returns (every 4 hours).
* **Cleaner desktop launch.** No more "Standalone" suffix on the
  desktop icon; opens a normal browser tab via the system default
  browser (Chrome / Edge / Safari / Firefox).
* **Bingo simplified** — traditional number bingo only; the music-video
  flow is gated behind a feature flag until that add-on ships.
* **Scheduling tool incorporated** — full host/employee scheduling +
  admin views are live in the desktop app.

## Important: macOS Gatekeeper

These Mac builds are **unsigned developer previews** because we don't
yet have an Apple Developer ID enrolled. The first time you open the
`.app` macOS will warn "BIG Hat Standalone is damaged and can't be
opened" or "cannot be opened because the developer cannot be
verified". To bypass:

```bash
# After dragging the app to /Applications:
xattr -dr com.apple.quarantine "/Applications/BIG Hat Standalone.app"

# Then double-click it normally.
```

Signed + notarized builds will replace these as soon as the Apple
Developer ID is provisioned. No re-purchase required — just download
the next release.
